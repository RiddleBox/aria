## 一、第一性原理：核心问题是什么

**人与 AI 协作时存在信息损失。**

手动输入、截图、复制粘贴——这些动作都在"我发现有价值的东西"和"AI 处理它"之间制造了摩擦和损耗。结果是：要么懒得记，要么记了但上下文丢了，要么事后找不到。

**ARIA 要解决的根本问题：**

> 让 AI 能"见我所见"，在我电脑使用的任意场景中，以最小摩擦捕捉我想保留的信息和观点。

---

## 二、核心交互模型

### 触发

```
快捷键按下 → 开始说话 → 松开/静音自动识别
```

- 快捷键是高优先级实现方式（游戏场景中键盘/鼠标更可靠）
- 唤醒词作为次级方式（日常工作场景，手不在键盘时）
- 早期也允许打字输入（降低测试门槛）

### 意图优先，感知按需

```
我说的话（指令）
    │
    ▼
AI 理解意图
    │
    ├── 需要视觉上下文？→ 截图 / 触发 Replay Buffer
    ├── 不需要？→ 直接处理
    │
    ▼
执行对应动作（归档 / 回答 / 转换 / ...）
    │
    ▼
语音回应我
```

**关键原则：截图和录屏不是默认行为，而是 AI 根据指令内容判断是否需要的上下文工具。**

### 典型交互示例

| 我说的话 | AI 判断 | 执行动作 |
|---|---|---|
| "帮我记这段，这个设计很有意思" | 需要视觉上下文 | 截图 + 前15s视频 + 我的评论 → 归档 |
| "这个机制是怎么运作的" | 需要视觉上下文 | 截图 → 分析 → 语音回答 |
| "把刚才那段转成GIF" | 需要视频 | 触发 Replay Buffer → 转 GIF → 告诉我路径 |
| "帮我想个游戏角色名字" | 不需要截图 | 直接回答 |
| "今天的简报有什么值得关注的" | 不需要截图 | 读 Obsidian 简报 → 语音摘要 |

## 三、设计原则

### 1. 模块化，核心不耦合具体能力
- Core 只负责：**听懂 → 判断意图 → 路由 → 调用**
- 新增能力 = 新增模块文件，不改核心代码
- 场景扩展（游戏 → 工作 → 智能眼镜）也通过模块实现

### 2. 奥卡姆剃刀：够用就好，不过早复杂化
- 存储：先用 Obsidian（已有，够用）
- 语音识别：本地 Whisper（离线、免费、够用）
- TTS：edge-tts（免费、中文够用）
- 只有明确不满足时才引入新依赖

### 3. 隐私友好
- 感知是被动触发，不持续上传屏幕
- 所有数据默认本地存储
- AI 调用按需，不持续消耗 token

## 四、模块规划

### Actions（AI 能做什么）

| 模块 | 功能 | 阶段 |
|---|---|---|
| `capture` | 截图 + OBS Replay Buffer 录屏 | Phase 1 |
| `archive` | 结构化写入 Obsidian md | Phase 1 |
| `convert` | 视频→GIF（ffmpeg）| Phase 1 |
| `answer` | 基于截图内容回答问题 | Phase 1 |
| `browse` | 打开 / 摘要网页内容 | Phase 2 |
| `remind` | 设定提醒 | Phase 2 |
| `search` | 在已归档内容中检索 | Phase 2 |

### Identity（AI 是什么）

| 模块 | 功能 | 阶段 |
|---|---|---|
| `persona` | 性格、记忆、对话风格 | Phase 1（基础版）|
| `voice` | TTS 语音输出 | Phase 1 |
| `avatar` | 虚拟形象（屏幕角落小角色）| Phase 3 |

### Perception（AI 怎么感知）

| 模块 | 功能 | 阶段 |
|---|---|---|
| 快捷键触发 | 全局热键监听 | Phase 1 |
| 语音识别 | Whisper 本地识别 | Phase 1 |
| 唤醒词 | 常驻监听 | Phase 2 |
| 场景感知 | 本地轻量视觉模型，主动检测变化 | Phase 2 |
| 硬件接入 | 智能眼镜等外部设备 | Phase 3 |

## 五、分阶段路线图

### Phase 1 · MVP：核心交互跑通（当前目标）

**目标：** 快捷键按下 → 说话 → AI 判断意图 → 执行 → 语音回应

**验收标准：**
- [ ] 全局快捷键可以在游戏中触发（不被游戏拦截）
- [ ] 说"帮我记这段"→ 截图 + 视频保存 + 归档到 Obsidian，5秒内完成
- [ ] 说"这是什么意思"→ 截图 → AI 分析 → 语音回答
- [ ] 说"帮我想个名字"→ 直接回答，不截图
- [ ] 所有动作有语音确认反馈

**依赖环境：**
- OBS 已安装并配置 Replay Buffer（建议30秒）
- ffmpeg 已安装
- OpenAI API key（意图解析 + 视觉分析）

### Phase 2 · 扩展：更自然，更主动

- 唤醒词支持（适合非游戏场景）
- 本地轻量视觉感知（屏幕场景变化检测）
- 更多 Action 模块（搜索、提醒、浏览）
- 日常工作场景模块（会议、阅读、视频）

### Phase 3 · 未来：真正的"AI 见我所见"

- 虚拟形象（有存在感的伴侣）
- 智能眼镜硬件接入
- 主动感知 + 主动提醒

## 六、技术约束与边界

- **运行环境**：Windows（PowerShell），需支持全局热键
- **存储**：Obsidian vault，路径可配置
- **不做的事**（Phase 1）：
  - 不做 web 界面
  - 不做持续屏幕监控
  - 不做云同步（Obsidian Git 已解决）
  - 不做虚拟形象

## 七、技术决策记录

| 问题 | 决策 | 理由 |
|---|---|---|
| 全局热键按键 | `Ctrl+\`` | 游戏场景少占用，位置顺手；鼠标侧键等自定义键位配置文件支持 |
| Phase 1 视频录制 | 不做，只截图 | 录视频方案待专项调研，OBS 太笨重，Phase 1 截图够用 |
| Obsidian vault | 新建独立 vault，路径 `D:\AIproject\aria-vault` | 与 game-knowledge-base 解耦 |
| 项目名称 | ARIA（AI Runtime Interface Agent）| ✅ 确认 |

## 八、开放问题（待后续决策）

1. **录视频方案**（Phase 2 专项调研）：OBS 太笨重、画质控制差，需调研更轻量的替代方案。候选方向：
   - `ffmpeg` 直接捕获屏幕（无需第三方软件，但 Windows 下配置略复杂）
   - `D3D / DXGI Desktop Duplication API`（Windows 原生，低开销，游戏兼容性好）
   - `obs-cli` 无界面模式（OBS 去掉 UI，只保留录制能力）
   - 待专项调研后决策
